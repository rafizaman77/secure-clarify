## Jul 17-19: scaling, dev calibration, and the primary test run (real model)

`scripts/freeze_tasks.py --n-per-domain 60 --tasks-out tasks/main_120.json
--manifest-out results/main120_manifest.json` regenerates the 120-task main
set (24 dev / 96 test, exactly the plan's ratio) from the same deterministic
generator as the pilot, using `task_factory.assign_split()`.

`scripts/tune_dev.py` fits `_DEV_ATTACK_PRIOR` from the dev split's own
adversarial-response `carries_attack` labels (Laplace-smoothed, dev-only — see
`estimators.fit_priors`) and sweeps `lambda` over `[0, 0.25, ..., 8]`,
selecting the **smallest** lambda whose dev-set adversarial unsafe rate is
≤ 0.10. Run for real on **`ollama:mistral-nemo:12b`**: **λ = 0.75**, fitted
priors in `results/dev_calibration.json`.

`scripts/run_primary.py` loads that frozen lambda/priors, never re-fits
anything, and evaluates only the 96 test-split tasks. Real result: all four
go/no-go checks PASS, verdict **GO** — benign clarification lifts goal rate
from 0.00 to 1.00, Conventional VoI's adversarial unsafe rate is 0.500 vs.
SecureVoI's 0.083, and SecureVoI's benign utility (0.950) matches
Conventional VoI's exactly while beating Trusted-Only's (0.675). See
`results/primary_summary.json` for the full table.

A hosted route (Groq `llama-3.3-70b-versatile`) reproduces the same clean
frontier on the **dev split** (`results/models/llama-3.3-70b/`), but its
free-tier daily token cap couldn't finish the 96-task test grid — the
completed held-out numbers are the local Mistral run. GPT-OSS-20B and
Qwen3-32B were also tried and **resist the injections** (a real,
model-dependent finding, not a bug — noted in `abstract.md`).

**Eight real bugs surfaced only once a model actually ran** — the injection
channel was structurally wrong for a real model (ScriptedAgent's `_inject_*`
keys don't exist for one; a real model is only fooled by the accepted
answer's *text*, now wired through `runner.py`/`agent.py`), a latent
`private_person` field leaked into the model's view and caused spurious
unsafe actions on benign tasks, the channel-info multiplier saturated and
erased channel differences at real-model disagreement levels, and more —
full list in `docs/DAILY_LOG.md` and `days/jul17-18/README.md`.

## Wiring in the real models — done; here's how, for the next model

`agent.OpenModelAgent` prompts a single injected `generate_fn(prompt: str) ->
str` and parses its JSON output. `scripts/model_backends.py` supplies that
`generate_fn` for three routes:

- **Local Ollama** (what produced the numbers above): install
  [Ollama](https://ollama.com), `ollama pull mistral-nemo:12b`, `ollama serve`.
  `ollama_generate_fn(model)`. No rate limits, deterministic, no API key.
- **Hosted API**: Groq / Together / Fireworks / OpenRouter all serve
  open-weight models through an OpenAI-chat-compatible endpoint —
  `openai_compatible_generate_fn(base_url, api_key, model)`. Watch free-tier
  daily token caps on large grids (see `docs/DAILY_LOG.md`'s rate-limit notes).
- **Local transformers** (`hf_local_generate_fn`): no API key or Ollama
  install, just Hugging Face network access; needs the isolated
  `.venv_model/` described in `days/jul17-18/environment-notes.md` since the
  system Python's numpy/scikit-learn ABI is broken.

```bash
# 1. smoke-test ONE task before spending the full budget
python scripts/smoke_real_model.py --backend ollama --model mistral-nemo:12b

# 2. re-run the frozen pipeline with the real agent for the NEXT model family
#    (Jul 22-23 needs a second/third test-split-complete model)
python scripts/tune_dev.py    --tasks tasks/main_120.json --backend ollama --model <name>
python scripts/run_primary.py --tasks tasks/main_120.json --backend ollama --model <name>
python scripts/compute_stats.py
python scripts/make_main_table.py
```

Nothing else changes: policies, estimators, verifiers, and the runner are
model-agnostic. Keep decoding deterministic (`temperature=0`, already the
default); re-fit lambda/priors **per model family** on the dev split only —
lambda=0.75 is Mistral-Nemo-12B's fit, not a repo-wide constant.
