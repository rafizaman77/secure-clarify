## Jul 17-19: scaling, dev calibration, and the primary test run

`scripts/freeze_tasks.py --n-per-domain 60 --tasks-out tasks/main_120.json
--manifest-out results/main120_manifest.json` regenerates the 120-task main
set (24 dev / 96 test, exactly the plan's ratio) from the same deterministic
generator as the pilot, using `task_factory.assign_split()`.

`scripts/tune_dev.py` fits `_DEV_ATTACK_PRIOR` from the dev split's own
adversarial-response `carries_attack` labels (Laplace-smoothed, dev-only — see
`estimators.fit_priors`) and sweeps `lambda` over `[0, 0.25, ..., 8]`,
selecting the **smallest** lambda whose dev-set adversarial unsafe rate is
≤ 0.10 (not the lambda that maximizes dev-set utility — on a 24-task split,
utility vs. lambda isn't guaranteed monotonic, only unsafe-rate is, so hill-
climbing utility risks latching onto sampling noise at a large, fragile
lambda). Current result: **λ = 0.25**, fitted priors in
`results/dev_calibration.json`.

`scripts/run_primary.py` loads that frozen lambda/priors, never re-fits
anything, and evaluates only the 96 test-split tasks. Current result (still
ScriptedAgent): all four go/no-go checks PASS, verdict **GO** — benign
clarification lifts goal rate from 0.00 to 1.00, Conventional VoI's
adversarial unsafe rate is 0.417 vs. SecureVoI's 0.042, and SecureVoI's benign
utility (0.921) still beats Trusted-Only (0.637). See
`results/primary_summary.json` for the full table.

## Jul 20: bootstrap statistics and the main table

`scripts/compute_stats.py` bootstraps 95% CIs (2000 resamples, task-level —
not episode-level, since all 4 policies share the same 96 test tasks and are
therefore paired) for every policy × condition, plus two paired-difference
significance tests. Both come out significant on the current (ScriptedAgent)
data: SecureVoI vs. Conventional VoI's adversarial unsafe rate, **−0.375
[−0.469, −0.281], p<0.001**; SecureVoI vs. Trusted-Only's benign utility,
**+0.283 [0.197, 0.371], p<0.001**. `scripts/make_main_table.py` renders this
as `results/main_table.md`, ready to paste into the paper once the inputs are
from a real model. Neither script needs to change when that happens — only
the `results/primary_episodes.json` they read does.

## Wiring in the real models

`agent.OpenModelAgent` is implemented: `sample_intents`, `classify_malice`, and
`act` all prompt a single injected `generate_fn(prompt: str) -> str` and parse
its JSON output (failing safe/closed on malformed responses — see the
docstrings in `secure_clarify/agent.py`). `scripts/model_backends.py` now
supplies that `generate_fn` for the two routes that actually fit this dev
environment (no GPU — CPU-only torch):

- **Hosted API** (recommended): Groq / Together / Fireworks / OpenRouter all
  serve open-weight models through an OpenAI-chat-compatible endpoint.
  `openai_compatible_generate_fn(base_url, api_key, model)`.
- **Local, free, CPU-friendly**: install [Ollama](https://ollama.com),
  `ollama pull llama3.1:8b`, `ollama serve`.
  `ollama_generate_fn(model)`.

```bash
# 1. smoke-test ONE task before spending the full budget
export GROQ_API_KEY=...
python scripts/smoke_real_model.py --backend openai \
    --base-url https://api.groq.com/openai/v1/chat/completions \
    --api-key-env GROQ_API_KEY --model llama-3.1-8b-instant

# 2. re-run the frozen pipeline with the real agent (repeat per model family --
#    the plan needs 2 for the pilot re-run, 3 for the main experiment)
python scripts/tune_dev.py    --tasks tasks/main_120.json
python scripts/run_primary.py --tasks tasks/main_120.json
```

Nothing else changes: policies, estimators, verifiers, and the runner are
model-agnostic. Keep decoding deterministic (`temperature=0`, already the
default in `model_backends.py`); re-fit lambda/priors **per model family** on
the dev split only — see `docs/DAILY_LOG.md`'s "What's actually needed to turn
these into real numbers" section for the full walkthrough and reasoning.
