generally, which belongs in the paper's discussion, not just a "safer model"

## Two real bugs found running this against actual cloud usage (not ScriptedAgent)
1. **`ollama_generate_fn`'s retry logic was local-server-flaky-tolerant, not
   rate-limit-aware** — `max_retries=2` with a flat 2s sleep gave up almost
   immediately on a genuine HTTP 429, instead of reading `Retry-After` the
   way the Groq/OpenAI-compatible path already did. Fixed to match.
2. **`run_full_model.py`'s `make_main_table` step passed no arguments** —
   `make_main_table.py` was hardcoded to the shared `results/stats.json`, so
   it silently kept re-rendering Mistral-Nemo's table instead of each new
   model's own. Added `--stats`/`--out` args to `make_main_table.py`, wired
   them through the orchestration, and regenerated the two missing tables
   directly.
